mappedBy는 다대다 관계 매핑시 사용한다.

@OneToMany(얘는 리스트로 관리) / @ManyToOne / @OneToOne

mappedBy는

```java
public class UserEntity {

    @Id
    @Column(name = "user_id", unique = true, nullable = false)
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @OneToMany(mappedBy = "userEntity", cascade = CascadeType.ALL, orphanRemoval = true)
    @JsonManagedReference
    private List<BoardEntity> boards = new ArrayList<>();

}

public class BoardEntity {

    @Id
    @Column(name = "board_id", nullable = false, unique = true)
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id")
    @JsonBackReference
    private UserEntity userEntity;

}
```

```java
@Entity
class Board{
	
	@Id
	private Long id;
	private String content;
	private Date createdAt;
	
		//mappedBy 바로 다음에 나오는건, 참조하는 쪽의 필드명이다.
	@OneToMany(mappedBy = "board", cascade = CascadeType.ALL)
	private List<Comment> comments = new ArrayList<>();
}

@Entity
class Comment {
	
	@Id
	private Long id;
	private String content;
	
	//이걸하면 얘를 조회때마다 조회목록에 안들어오게 함(메모리 이득)	
	@ManyToOne(fetch = FecthType.LAZY)
	
	//이곳은 지금 FK이므로, 참조 하는 쪽의 이름을 적는 거다.
	@JoinColumn(name = "board_id") // board_id는 실제 DB 컬럼명
		private Board board;

}
```

- **부모(Owner, “주인”) 쪽**에는 `@JsonManagedReference`
- **자식(참조, “Back”) 쪽**에는 `@JsonBackReference`
