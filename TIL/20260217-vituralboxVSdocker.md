# VirtualBox vs Docker 차이점 정리
둘 다 가상화 기술이라는 점은 같지만, "어디까지 가상화하느냐"에 따라 무게감과 용도가 완전히 달라지는 개념입니다.
## 1. VirtualBox (Virtual Machine)
"컴퓨터 안에 진짜 컴퓨터를 한 대 더 만드는 것"
핵심 개념: 하드웨어 수준의 가상화.
구조: Host OS 위에 Hypervisor(하나의 물리적 컴퓨터에서 여러 개의 운영체제(OS)를 동시에 실행할 수 있게 해주는 가상화 관리 소프트웨어)를 두고, 그 위에 독립된 Guest OS(Windows, Linux 등)를 통째로 설치합니다.
특징: OS 전체를 부팅해야 하므로 무겁고 느리다.
하지만 OS 자체가 격리되어 있어 보안이 강력하고, 서로 다른 OS를 실행할 수 있습니다.

## 2. Docker (Container)
"내 컴퓨터의 엔진(커널)은 공유하고, 실행 환경만 격리하는 것"
핵심 개념: 운영체제(OS) 수준의 가상화.
구조: Host OS의 커널(Kernel)을 공유하며, 앱 실행에 필요한 라이브러리와 실행 파일만 컨테이너로 묶어서 실행합니다.
특징: OS 부팅이 없어서 매우 빠르고 가볍다.
